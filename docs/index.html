How it works

One pipeline.
From voice to document.

The doctor speaks naturally. Every other step is handled automatically.

01
Doctor joins a LiveKit room
The browser connects to a LiveKit room and begins streaming audio. No additional hardware required — just a microphone.
LiveKit WebRTC
02
The agent listens silently
A Python worker subscribes to the audio track and forwards every frame to Deepgram's medical-grade speech-to-text model. It never speaks, never interrupts.
Deepgram nova-2-medical
03
Transcript streams live to the UI
Each recognised segment is pushed via WebSocket to the React frontend in real time. The doctor can watch the transcription build as they speak.
FastAPI · WebSocket
04
Note is generated on session end
The full transcript is sent to Groq's LLaMA 3.3 70B model with a strict clinical prompt. It extracts symptoms, diagnosis, medications with dosages, precautions, and follow-up into a typed JSON structure.
Groq · LLaMA 3.3 70B
05
Doctor reviews and downloads
The structured note renders instantly on the right panel. One click exports a professionally formatted PDF or DOCX ready for review, signature, and filing.
ReportLab · python-docx

Features

Built for real clinical workflows.

Every design decision was made with a real consultation in mind.

🎙

Purely observant agent

The agent never speaks, asks questions, or interferes. It exists only to listen — hands off, always.

⚡

Medical-grade transcription

Deepgram's nova-2-medical model is trained on clinical speech, handling drug names, dosages, and symptoms accurately.

📋

Structured SOAP notes

Chief complaint, symptoms, observations, diagnosis, medications, precautions, healthy practices, and follow-up — all extracted automatically.

🔒

No hallucinated dosages

The prompt explicitly instructs the model to leave dosage fields null if not stated. Nothing is inferred or invented.

📄

PDF & DOCX export

One-click download of a professionally formatted note with headers, medication tables, and a physician sign-off disclaimer.

🌐

Real-time split panel UI

Live transcript streams on the left, structured note builds on the right. Recording timer, generation spinner, export buttons — all with distinct states.

Tech stack

Every layer chosen deliberately.

No bloat. Each tool does exactly one job and does it well.

Audio streaming
LiveKit
WebRTC room management. Real-time low-latency audio without building infrastructure from scratch.
Speech-to-text
Deepgram nova-2-medical
Purpose-built for clinical vocabulary. Dramatically better accuracy on drug names and medical terminology.
Note generation
Groq · LLaMA 3.3 70B
Fast inference. Strict JSON output mode ensures the note is always parseable and never hallucinated.
Backend
FastAPI · Python
Token issuance, WebSocket broadcasting, note generation orchestration, and file export endpoints.
Frontend
React · Vite
Live split-panel UI. Transcript streams in real time via WebSocket. Export buttons with distinct loading states.
Document export
ReportLab · python-docx
Professionally formatted PDF and DOCX output with clinical headers, medication tables, and sign-off disclaimer.
